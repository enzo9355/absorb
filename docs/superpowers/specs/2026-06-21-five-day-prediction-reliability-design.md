# 五日預測可靠度改善設計

## 目標

在不新增大型框架或部署服務的前提下，讓 LINE Bot 顯示的機率具有一致、可重現的定義：

> 目前收盤後，下一個第五個交易日的收盤價高於目前收盤價的機率。

本階段先提高資料與驗證的可信度，不承諾特定命中率。

## 現有問題

- 最後五筆資料沒有未來價格，卻被轉成下跌標籤。
- 模型預測五日方向，回測使用次日報酬，目標不一致。
- 單次 80/20 切割容易受特定行情影響，且沒有五日隔離區。
- FinMind 股價資料中的成交量被丟棄。
- 新聞關鍵字分數直接加減模型機率，破壞機率校準。
- Yahoo Finance 備援只嘗試上市股票的 `.TW`，上櫃股票可能取不到資料。

## 設計

### 資料

- 保留 FinMind `TaiwanStockPrice` 的 OHLCV；Yahoo Finance 備援也保留 `Volume`。
- Yahoo Finance 備援依序嘗試 `.TW` 與 `.TWO`。
- 額外讀取 FinMind：
  - `TaiwanStockInstitutionalInvestorsBuySell`
  - `TaiwanStockMarginPurchaseShortSale`
- 額外資料失敗時仍使用 OHLCV 分析；對應特徵設為中性值，不讓整個查詢失敗。
- 沿用現有一小時分析快取，不加入資料庫或新快取服務。

### 特徵

保留現有 MA、RSI、MACD、KD 與波動率，增加少量可解釋特徵：

- 5、20 日報酬
- 當日振幅
- 5 日／20 日成交量比
- 成交量日變化
- 三大法人淨買賣占成交量比
- 融資餘額與融券餘額變化率

所有比例分母加上極小值並處理無限值，避免低成交量股票造成模型失真。

### 標籤與訓練

- `future_return_5d = Close.shift(-5) / Close - 1`
- 只在 `future_return_5d` 有值時建立訓練標籤；最後五筆必須排除。
- 使用既有 `sklearn.model_selection.TimeSeriesSplit` 做五折向前驗證，設定 `gap=5`。
- 每折只用過去資料訓練、未來資料驗證；類別不足的折跳過。
- 樣本外預測用於回測與歷史機率圖。
- 完成驗證後，以全部已知標籤重新訓練一次，僅預測最新一日。
- LightGBM 保持不變，不新增 LSTM、Transformer、Qlib 或強化學習依賴。

### 回測與顯示

- 回測報酬改為與標籤一致的五日報酬。
- 為避免重疊持有期重複計算，每五個交易日最多採用一次訊號。
- 進場門檻維持 60%。
- 每次完整買賣預扣 0.585% 成本，作為台股手續費與交易稅的保守基準。
- 新增或修正以下樣本外指標：
  - 五日方向準確率
  - Brier Score
  - 60% 門檻訊號勝率與次數
  - 扣除成本後累積報酬
  - 最大回檔與夏普值
- LINE 與網頁上的「AI 勝率」改稱「五日上漲機率」。
- 新聞情緒保留獨立展示，不再直接修改模型機率。

### 錯誤處理

- 價格資料不足、時間切割後沒有有效驗證折、或模型只有單一類別時，回傳既有的「資料暫時無法取得」流程。
- 籌碼資料錯誤時將相關特徵設為中性值，不中止價格分析，也不改變模型欄位。
- 外部請求保留逾時；不使用裸 `except` 吞掉新加入流程的錯誤，至少記錄來源與錯誤訊息。

## 驗證

新增一個小型 `unittest` 測試檔，使用合成 OHLCV 資料檢查：

1. 最後五筆不會進入訓練標籤。
2. 每個驗證區段都晚於訓練區段，且保留五日 gap。
3. 回測採用五日報酬並扣除交易成本。
4. 外部籌碼資料缺失時仍能產生 OHLCV 特徵。

另外執行 Python 語法編譯檢查；若環境憑證可用，再以一檔上市與一檔上櫃股票做即時 smoke test。

## 不在本階段範圍

- 跨股票共同模型
- 月營收與財務報表的 point-in-time 合併
- 自動排程訓練與模型檔儲存
- Qlib、vectorbt、skfolio、FinRL 或深度學習模型
- 自動下單與投資組合最佳化

這些項目只在本階段的樣本外結果證明有效後再評估。

## 完成條件

- 測試與語法檢查通過。
- 標籤、驗證、回測均使用相同的五日預測期間。
- 最後五筆未知標籤不參與訓練。
- 對外資料缺失不會讓 OHLCV 分析失效。
- 現有 LINE 指令與 Flask 路由維持相容。
