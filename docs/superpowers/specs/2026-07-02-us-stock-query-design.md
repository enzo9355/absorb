# 美股個股查詢設計

## 目標

讓 LINE 與 Web 可直接查詢標準美股代碼，例如 `AAPL`、`NVDA`、`MSFT` 與 `SPY`，並沿用現有五日預測、回測、技術指標、新聞情緒、關注及提醒功能。

## 資料與模型

- 美股 OHLCV 使用專案既有 `yfinance`，不新增套件或 API 金鑰。
- 美股市場特徵使用 `^GSPC` 與 `SPY`；台股仍使用 `^TWII` 與 `0050.TW`。
- 美股沒有 FinMind 法人、外資及融資融券資料，相關特徵維持中性，介面顯示資料不足。
- 模型仍對單一股票使用時間序列切分訓練，不共用台股模型權重。

## 輸入與輸出

- 支援 1 至 5 個英文字母的標準美股代碼，不支援選擇權、期貨、虛擬貨幣與含符號的特殊股別。
- 美股顯示名稱先使用 `美股 <ticker>`，避免額外呼叫慢速公司資訊 API。
- `/stock/<ticker>`、LINE 查詢、試算、關注與提醒沿用既有流程。

## 限制

- 本階段不做全美股每日掃描或美股產業榜，避免超過 1GB Cloud Run、Yahoo 限流與排程時間。
- 無效代碼會由行情空結果安全降級為「查無資料」。

## 驗證

- 測試代碼解析、台美股資料分流、市場基準、Web 路由及既有台股回歸。
- 以 AAPL 執行實際資料與分析煙霧測試，再部署 Cloud Run。
