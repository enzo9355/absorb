# 產業候選股預篩與關鍵字查詢設計

## 目標

提升每日產業預測的真實市場代表性，並讓使用者不必只能點選 quick reply。使用者可以直接輸入產業關鍵字，例如 `半導體`、`AI`、`航運`、`金融`、`ETF`，LINE Bot 會回覆對應的每日產業預測。

## 不做的事

- 不新增大型套件。
- 不在 LINE webhook 內即時掃描多檔股票。
- 不新增市值資料源。現階段用成交金額近似市場關注度，避免多一個外部資料故障點。
- 不改 LightGBM 模型核心。

## 現況

目前每日排程流程是：

```text
twstock 產業分類
  -> 每個產業取前 20 檔
  -> 對 20 檔跑 analyze()
  -> 存入 Firestore sector_signals 快照
  -> LINE 選產業時讀快照
```

問題是「前 20 檔」仍偏靜態，不能反映當天市場活躍度。

## 新流程

改成兩段式：

```text
twstock 產業分類
  -> 用便宜市場資料預篩候選股
     - 近 20 日平均成交量
     - 近 5 日外資買賣超
     - 近 20 日平均成交金額：收盤價 × 成交量
  -> 每個產業取預篩分數前 20 檔
  -> 對 20 檔跑完整 analyze()
  -> 存入 Firestore sector_signals 快照
  -> LINE 產業查詢讀快照
```

## 候選股預篩分數

預篩只用現有資料源與現有欄位：

```text
candidate_score =
  成交量分數
  + 成交金額分數
  + 外資買超分數
```

規則：

- 成交量與成交金額使用近 20 日平均。
- 外資買賣超使用近 5 日加總。
- 分數只用於挑出要跑完整分析的 20 檔，不直接對使用者顯示。
- 若某檔預篩資料抓取失敗，保留原本順序作為 fallback。

這樣可以讓產業預測更靠近「市場現在正在交易與關注的股票」，但仍控制每天完整分析的數量。

## LINE 關鍵字查詢

使用者仍可使用：

```text
預測 -> quick reply -> 選產業_<產業名>
```

新增支援直接輸入：

```text
半導體
AI
伺服器
金融
航運
ETF
生技
電動車
```

處理順序：

```text
使用者文字
  -> 先判斷提醒/關注/試算等既有指令
  -> 再判斷股票代碼或股票名稱
  -> 再判斷產業關鍵字
  -> 找到單一產業：回覆每日產業預測
  -> 找到多個產業：回覆候選產業按鈕
  -> 找不到：維持原本提示
```

股票查詢優先於產業查詢，避免使用者輸入公司名稱時被誤判成產業。

## 產業別名

新增一份輕量別名字典，不接外部服務：

```json
{
  "AI": ["AI伺服器", "電腦及週邊設備業", "電子零組件業"],
  "伺服器": ["AI伺服器"],
  "金融": ["金融保險業"],
  "半導體": ["半導體業"],
  "ETF": ["ETF專區"],
  "航運": ["航運業"],
  "生技": ["生技醫療業"],
  "電動車": ["汽車工業", "電機機械"]
}
```

若別名對應多個產業，LINE 回覆候選按鈕讓使用者選，不直接猜。

## 錯誤處理

- 預篩資料抓取失敗：該股票分數為 0，仍可按原本順序補足候選。
- 整個產業預篩失敗：退回原本前 20 檔。
- 使用者輸入產業但快照尚未產生：回覆「產業資料尚未更新」。
- 關鍵字對應多個產業：回覆 quick reply，不跑分析。

## 測試

新增最小測試：

- 預篩會優先挑成交量、成交金額、外資分數較高的股票。
- 預篩失敗時仍回傳最多 20 檔候選。
- 使用者直接輸入單一產業關鍵字時，LINE 讀取快照並回覆 Flex。
- 使用者輸入模糊關鍵字時，LINE 回覆候選產業按鈕，不呼叫 `analyze()`。
- 股票查詢優先於產業關鍵字。

## 成功標準

- 每個產業仍最多只跑 20 檔完整分析。
- LINE webhook 不同步掃描多檔股票。
- 使用者可以直接輸入產業關鍵字。
- 不新增依賴。
- 完整測試通過後才 push。
