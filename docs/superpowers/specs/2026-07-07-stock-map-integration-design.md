# Stock Papi 市場地圖整合設計

## 目標

將 AI 智慧產業地圖公開呈現的功能概念，轉化為 Stock Papi 自有的市場探索體驗。整合不得抓取或重製對方受限制的 API、資料與視覺資產，也不得破壞既有 LINE-first、Cloud Run 1GB、低冷啟動與本地量化快照架構。

## 公開功能盤點

公開首頁與公司頁可見功能分成六組：

1. 每日焦點：市場新聞、重大資訊、法人、資券、強勢股與大戶動向。
2. 題材與產業探索：題材漲跌排名、台股／美股／日股、產業鏈、ETF 與產業地圖。
3. 市場熱力圖：用顏色、漲跌與公司數呈現市場強弱。
4. 公司研究：基本資料、產業分析、財務、籌碼、ETF 持倉、技術、新聞與研究圖表。
5. 個人化：收藏、通知、AI 分析與研究報告。
6. 跨市場：台美日供應鏈聯動。

Stock Papi 已具備搜尋、台美股分析、五日機率、技術指標、回測、外資、新聞情緒、產業預測、LINE 關注與提醒、本地全市場快照。因此不複製整套產品，只補足「探索資訊架構」與尚未顯示的關聯資料。

## 方案比較

### A. 只改畫面

重新排列既有卡片並加熱力圖，速度最快，但公司與產業之間仍無關聯，提升有限。

### B. 原生漸進式整合（採用）

第一階段重用現有 Dashboard API、產業快照與個股分析資料，增加每日焦點、題材熱力圖、公司分頁導覽與產業同儕。第二階段才由本地批次補上 MOPS、財務、ETF 持倉與跨市場供應鏈。此方案不新增大型套件，也不讓 Cloud Run 全市場即時計算。

### C. 完整鏡像外部網站

需要大量資料源、帳號、付費與內容維護，也有著作權、robots.txt、資料授權與 1GB 資源風險，不採用。

## 第一階段設計

### Dashboard

- 保留搜尋與市場摘要。
- 新增「今日焦點」，直接使用現有精選標的與產業預測，不新增網路請求。
- 新增「市場熱力圖」，由既有產業卡的平均機率、趨勢與候選數計算顏色、強度與排序。
- 產業卡可直接進入代表標的分析，維持新手可理解的白話標籤。
- LINE 關注與提醒仍只在 LINE 管理，Web 不新增登入或第二份狀態。

### 個股頁

- 將長頁面整理成可跳轉的「總覽、預測、籌碼、情緒、新聞」導覽，不隱藏原有內容，也不依賴 JavaScript 才能閱讀。
- 新增「產業同儕」區塊，從現有產業分類找出同類標的並連往個股頁。
- 保留價格、回測、投資試算、外資、技術、情緒與新聞原始資料來源標示。

### 資料流

```text
現有產業快照／本地全市場快照
        ↓
dashboard_sector_cards()
        ↓
今日焦點 + 熱力圖 + 產業卡

build_market_map() + 個股代碼
        ↓
industry_peers()
        ↓
個股頁產業同儕
```

所有衍生資料為純函式計算，不新增資料庫、不在模組載入時發出請求。

## 第二階段資料

- MOPS 重大資訊：在本地批次抓取、去重與分類，發布摘要快照。
- 財務趨勢：營收、獲利、股利等欄位在本地計算後上傳。
- ETF 持倉：僅使用授權清楚的官方或公開資料源。
- 跨市場供應鏈：以自有分類表與公開公司資料建立關係，不複製外站的產業圖。

第二階段資料沿用目前的 gzip、SHA-256、manifest、覆蓋率與過期回退機制。Cloud Run 每次只讀取使用者正在看的股票或產業，不載入全市場到記憶體。

## Secret Manager

- 六個敏感值各自使用獨立 Secret。
- Cloud Run 執行帳號只有 `roles/secretmanager.secretAccessor`。
- `GCP_PROJECT_ID` 與 `QUANT_SNAPSHOT_BUCKET` 為非敏感設定，維持一般環境變數。
- 新修訂版必須通過 `/health` 與個股頁 smoke test 才能視為遷移成功。
- 舊修訂版仍可能保留歷史明文設定，需在確認新修訂版穩定後刪除，外部服務憑證則應依供應商能力輪替。

## 錯誤處理

- Dashboard 衍生資料失敗時，保留現有市場摘要與搜尋，不讓整頁失效。
- 找不到產業同儕時，不顯示空卡片。
- 本地快照遺失、過期或驗證失敗時，沿用既有即時計算回退。
- 不因外部網站不可用而影響 Stock Papi；正式系統不呼叫 aistockmap.com。

## 測試與驗收

- 純函式測試熱力圖排序、顏色級距、空資料與同儕排除自身。
- Web 測試確認 Dashboard 與個股頁出現新區塊，既有搜尋與圖表仍存在。
- 完整單元測試、HTML smoke test、ShellWard 掃描、Cloud Run `/health` 與 `/stock/2330` 驗證。
- 行動版不得橫向溢出；互動控制需有可讀標籤與鍵盤焦點。
