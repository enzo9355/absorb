# Stock Papi 響應式網站與液態玻璃 UI/UX 設計

## 目標

把目前偏傳統財經站風格的 Web 介面，重整成一套同時適用桌機與手機的雙端體驗。

這次設計只處理網站視覺語言、版型、資訊層級與互動規則，不改動 LINE 原生追蹤與提醒的定位。LINE 仍是主要入口與操作端，網站只負責「更完整的分析、圖表與建議」。

## 設計範圍

- 首頁 / 儀表板的桌機與手機版體驗。
- 個股詳細頁的桌機與手機版體驗。
- 統一的字體、色彩、卡片、按鈕、標籤與圖表容器規則。
- 一套節制使用的液態玻璃系統。
- `Stock Papi` 的品牌人設視覺準則。
- Figma 檔案的交付清單與頁面結構。

## 不做

- 不重做 LINE 端的資訊架構。
- 不新增暗色模式。
- 不把整站所有卡片都改成玻璃效果。
- 不在本設計階段改動預測模型、資料來源或後端 API。
- 不加入需要重型前端依賴的視覺特效框架。

## 設計原則

- 先讓使用者一眼看懂「現在能查什麼、下一步按哪裡」。
- 先給結論，再給圖表，最後才給指標拆解。
- 色彩可以豐富，但金融關鍵語意仍維持穩定：漲跌保留台股紅綠。
- 圖表、數值表格、密集資料區優先可讀性，不為了視覺效果犧牲判讀速度。
- 液態玻璃只用在互動焦點層，不覆蓋核心資訊層。

## 產品定位

網站與 LINE 的職責分工如下：

- LINE：查詢入口、加入關注、提醒管理、產業預測入口。
- Web：完整分析、K 線圖表、新聞情緒拆解、白話建議與延伸閱讀。

這個分工要在版面上被清楚表現。網站不再承擔「像 App 一樣管理清單」的責任。

## 視覺方向

採用亮色基底，不走深色財經站風格。

- 背景：暖白底。
- 品牌輔助色：薄荷綠、杏桃色、薰衣草紫。
- 金融功能色：台股上漲紅、下跌綠，獨立於品牌色。
- 英文與大數字：`Lora`
- 中文標題與內文：`源明雲體`

整體感受應該是乾淨、明快、有記憶點，但不幼稚，也不做過度科技藍紫風。

## 液態玻璃系統

本次採「單鏡片系統版」，不做全站玻璃化。

### 使用原則

- 玻璃效果只放在互動焦點元件。
- 重點不是單純模糊，而是邊緣高光、折射感與厚度。
- 若瀏覽器或使用者偏好不適合透明效果，必須能直接降級成高質感半透明或實心卡片。

### 允許使用的位置

- 首頁主搜尋框。
- 首頁或個股頁的篩選切換列。
- 手機底部浮動操作條。
- 個股頁頂部摘要膠囊。

### 禁止使用的位置

- K 線圖本體。
- 任何密集數值表格。
- 長新聞列表。
- 產業排序表或回測表。

### 視覺規格

- 背景透明度：`rgba(255,255,255,0.18 ~ 0.24)`
- 邊框：半透明白邊，搭配淡色高光。
- 陰影：柔和，不做厚重漂浮。
- 動態：僅允許輕微高光位移與視差。

### 降級規則

- `prefers-reduced-motion`：關閉玻璃動效。
- `prefers-reduced-transparency`：改成實心淡色卡片。
- 不支援透明濾鏡時：保留圓角、邊框與陰影，取消折射感。

## 資訊架構

### 首頁 / 儀表板

首頁只保留六個區塊，避免過多功能並列：

1. `Hero Search`
   - 主搜尋框。
   - 常用快捷鍵，例如熱門股票、熱門產業、熱門關鍵字。
2. `Market Pulse`
   - 三張摘要卡，顯示市場情緒、產業熱度、今日關鍵訊號。
3. `Watchlist Strip`
   - 自選股橫向卡片列，含漲跌、提醒、是否有新訊號。
4. `Industry Forecast`
   - 首頁主功能，清楚取代原本模糊的強勢訊號印象。
5. `Top Picks`
   - 從產業預測挑出最值得看的標的與結論。
6. `Learn / Explain`
   - 新手白話解釋區，說明勝率、情緒、外資等指標。

### 個股詳細頁

個股頁遵循固定順序：

1. 結論摘要
2. 圖表與趨勢
3. 關鍵數據卡
4. 新聞情緒與輿論拆解
5. 白話建議與風險提醒

先讓使用者知道「現在怎麼看」，再決定要不要往下讀更多細節。

## 響應式版型

### 桌機版

- 基準寬度：`1440px`
- 主要內容寬度：`1180px`
- 左右留白：`80px`
- Grid：`12` 欄
- 區塊間距：`32px`
- 大區塊上下留白：`56px`

區塊配置：

- `Hero Search`：滿版主視覺，高度約 `280~320px`
- `Market Pulse`：三欄等寬
- `Watchlist Strip`：單列橫向卡片
- `Industry Forecast`：`2 x 3` 卡片網格
- `Top Picks`：左 `2/3`、右 `1/3`
- `Learn / Explain`：雙欄

### 手機版

- 基準寬度：`390px`
- 左右內距：`16px`
- 區塊間距：`20px`
- 大區塊上下留白：`28px`

區塊順序固定為：

1. 搜尋
2. 市場摘要
3. 自選股
4. 產業預測
5. 精選標的
6. 教學說明

`Industry Forecast` 在手機版改成單欄卡片，不做雙欄壓縮。

## 字體層級

- `Display`：`32/40`
- `H1`：`24/32`
- `H2`：`20/28`
- `H3`：`16/24`
- `Body`：`15/24`
- `Caption`：`12/18`

規則：

- 大數字、英文 ticker、重點指標可用 `Lora`。
- 中文長文一律用 `源明雲體`。
- 行高保持偏寬鬆，不做緊密擠壓式財經排版。

## 元件規則

### 卡片

- 一般資訊卡：暖白實底。
- 焦點卡：可使用液態玻璃。
- 圓角：`24px`
- 內距：`20~24px`

### 按鈕

- 主按鈕：薄荷綠實底。
- 次按鈕：半透明白底或淡色描邊。
- 取消 / 危險操作：不用大面積實心紅底，以文字與細邊框表達。

### 標籤

- 漲跌標籤維持紅綠。
- 品牌型標籤以薄荷、杏桃、薰衣草交替。
- 不用 emoji 輔助辨識。

### 圖表容器

- 圖表永遠包在實心卡片裡。
- 容器可有固定比例與內距。
- 圖表工具列可做玻璃化，但圖表本體不得透明。
- 手機版必須保證圖表不超出容器、不壓住其他數據。

## Stock Papi 人設視覺準則

`Stock Papi` 不是冷冰冰的券商後台，也不是賣萌助手。

視覺與文案要呈現的角色是：

- 專業，但不艱澀。
- 有判斷，但不裝懂。
- 給方向與結論，不堆砌過度華麗的修辭。

對應到設計上：

- 標題應直說功能，不玩過度抽象命名。
- 結論卡要像顧問摘要，不像社群貼文。
- 色彩雖豐富，但版面結構要克制、整齊。
- 首頁最醒目的區塊必須是搜尋與產業預測，不是裝飾圖。

## Figma 交付清單

Figma 檔案至少包含以下頁面：

1. `Foundations`
   - 色票
   - 字體階層
   - 間距與圓角規則
   - 液態玻璃規格
2. `Components`
   - 按鈕
   - 標籤
   - 一般卡片
   - 焦點玻璃卡
   - 搜尋框
   - 手機浮動操作條
3. `Desktop`
   - 首頁 / 儀表板
   - 個股詳細頁
4. `Mobile`
   - 首頁 / 儀表板
   - 個股詳細頁

## Figma 產出規則

- 先做框架與主要版型，再補元件細節。
- 桌機與手機版都要完整，不只縮放同一個畫面。
- 液態玻璃效果只建立成少數可重複使用的焦點元件，不擴散到所有卡片。
- `Industry Forecast` 必須在首頁中段成為明顯主區塊。
- 個股頁的圖表容器要明確標示安全邊界，避免後續實作再次出現溢出。

## 成功標準

- 使用者一進首頁就知道可以搜尋、看產業預測、看自選股。
- 手機版比目前更直覺，且主要區塊不需要橫向縮放。
- 整體色彩比既有版本更活，但不影響數值與圖表判讀。
- 液態玻璃成為品牌識別的一部分，但不變成閱讀負擔。
- LINE 與 Web 的分工被清楚表現。
